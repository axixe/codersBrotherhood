Все 4 вывода в консоль будут иметь Bad: undefined.
Причина этого поведения заключается в том, что переменная i при создании цикла имеет глобальную область видимости из-за объявления её как var. Спустя время, когда выводится в консоль arr[i] она уже не имеет никакого значения, т.к. цикл был завершён.
А Bad выводится из-за того, что undefined - это значение false, поэтому при переборе вариантов через строчное условие у нас выводится все Bad.

Исправить это поведение можно многими вариантами, но т.к. в условии было привести всего два варианта, то я расскажу про них:

1) Заменить var i = 0 на let i = 0:

const arr = [10, 12, 15, 21];

for (let i = 0; i < arr.length; i++) {
    setTimeout(function() {
        console.log(arr[i] > 13 ? `Good: ${arr[i]}` : `Bad: ${arr[i]}`);
    }, 3000);
}

2) Заменить сам цикл перебора массива на метод forEach, который присутствует у прототипа массива:

const arr = [10, 12, 15, 21];

arr.forEach((_, index) => {
    setTimeout(function() {
        console.log(arr[index] > 13 ? `Good: ${arr[index]}` : `Bad: ${arr[index]}`);
    }, 3000);
})
